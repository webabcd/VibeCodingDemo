# Vibe Coding Demo With Trae
- Trae 是基于 Visual Studio Code 开发的‌

### trae 的 ide 模式和 solo 模式
- ide 模式：类似于助手模式，所有步骤都会展现出来，相当于在传统的开发模式下集成了 ai 助手
- solo 模式：类似于代理模式，所有步骤 ai 都自己做了，中间的过程可以实时监测
- 点击 trae 的左上角可以切换 ide 模式和 solo 模式

### trae 的 solo 模式（SOLO Coder 智能体）
- 支持多任务并行，在 solo 模式的左侧面板可以创建新任务，重命名任务，删除任务。每个任务自己是串行的，多任务之间是并行的。
- 在 solo 模式的右侧面板，可以开启或关闭“实时跟随”
- “实时跟随”用于实时监测 ai 的工作成果
  - 其会自动打开“终端”展示命令的执行过程和结果
  - 其会自动打开“编辑器”显示编码过程、最终代码、被修改的代码等，开发者可以保留或撤销 ai 生成的代码
  - 等等

### trae 的 ide 模式
- 每个由 ai 生成的文件，开发者都可以选择保留或撤销
- 每个由 ai 修改的文件，开发者都可以查看新旧文件的差异，然后选择保留或撤销
- 如果 ai 生成或修改了多个文件，开发者可以选择全部保留或全部撤销
- 通过 Chat 智能体开发时，所有代码的修改都要开发者确认，命令的执行需要开发者手动执行
- 通过 Builder 智能体开发时，所有代码的修改都不需要开发者确认，命令的执行由智能体自动执行
- 右侧面板的右上角的 2 按钮，可以新建任务或者加载历史任务
- 右侧面板的对话记录中，开发者发送的每条消息的左边都有一个回退按钮，点击后可以将代码回退到此消息之前的状态
- 在对话框中输入 /clear 或 /reset 可以清空上下文

### trae 的智能体
- 内置智能体有 Chat, Builder, Builder with MCP, SOLO Coder
- Chat 一般用于与 ai 进行普通的对话，比如问一些问题，或者让 ai 实现某个功能，然后把 ai 生成的代码手动复制到代码编辑器中
- Builder, Builder with MCP, SOLO Coder 一般用于让 ai 根据需求开发，它会直接生成代码文件，或修改代码文件，或执行命令
- 自定义智能体：设置 -> 智能体 -> 创建，然后输入提示词并选择工具后就创建了一个自定义智能体
  - 比如要创建一个 Python 开发智能体，提示词可以让 ai 帮写，[提示词示例](prompt/agent_python.md)

### trae 的对话框（通过开发一个小游戏介绍 trae 的对话框的使用）
1. 通过 @ 选择智能体，切换至 solo 模式
   1. @SOLO Coder
2. 选择大模型
   1. Auto Mode 由 trae 自动选择大模型
   2. 也可以手动指定大模型，比如 GLM 或 Kimi 之类的
3. 通过“四角星光按钮”优化提示词
   1. 输入：新建 snake 目录，并在 snake 目录内开发一个基于 html 的贪吃蛇游戏
   2. 点击“四角星光按钮”优化提示词
   3. 发送
4. 通过 # 引用上下文
   1. #snake/todo.md 参考此文件修改游戏逻辑
   2. 运行游戏
5. 切换至 ide 模式，引用图片（solo 模式无法引用图片）
   1. @Builder
   2. 点击对话框的“图片按钮”，选择 snake/button.jpg
   3. 按照这个按钮的样式修改游戏中的所有按钮
   4. 运行游戏
6. 在对话框中支持的快捷键
   1. Enter：发送
   2. Shift + Enter：换行
   3. 上下：切换历史输入

### trae 的 CUE（Context Understanding Engine）和编辑器内对话
- CUE 用于在理解上下文后，实现代码补全、多行修改、修改点预测，修改点跳转等
  - 例如：可以先输入函数注释，然后通过 CUE 实现函数的功能
  - 例如：可以按照常规方式写代码，CUE 会自动预测接下来的代码
  - 例如：修改了代码的某一点，CUE 会自动提示需要修改的关联点
- 在编辑器中，CUE 会在理解上下文后，在你光标的位置启用 CUE 功能（通过输入字符或鼠标点击等都可能会唤起 CUE 功能）
- CUE 被唤起后，Tab 采纳，Escape 拒绝，Shift + Tab 逐行采纳，Ctrl + RightArrow 逐词采纳
- 通过 trae 的右下角的 CUE 图标，可以选择是否启用 CUE 功能
- 打开 python/weather.py 文件，测试并学习 CUE

### trae 编辑器内对话
- 通过 Ctrl + I 可以在光标位置唤起编辑器内对话框，对话时会自动发送上下文给大模型
  - 例如：描述函数的功能后，可以自动生成函数的代码
  - 例如：可以为函数添加注释
  - 例如：可以优化选中的代码，重构选中的代码
- 打开 python/weather.py 文件，测试并学习编辑器内对话

### trae 上下文
- 在编辑器或终端窗口中，选择一段内容，然后按 Ctrl + U 会将其添加到对话框的上下文中
- 在对话框中，过 # 可以添加多种类型的上下文，比如文件、目录、文档集等
- 设置 -> 代码索引，引用 #Workspace, #Folder 时，请先保证这个代码索引已经构建完成
- 设置 -> 忽略文件，引用 #Workspace, #Folder 时，不会将这些文件添加到上下文中（.gitignore 中定义的文件也不会被添加到上下文中）
- 设置 -> 文档集，用于管理自定义文档集，以便后续通过 # 引用
- 设置 -> 规则和技能，可以管理个人规则（提示词可以让 ai 帮写，[提示词示例](prompt/user_rule.md)）和项目规则（提示词可以让 ai 帮写，[提示词示例](prompt/project_rule.md)）

### trae 的小技巧
- 预览 markdown 文件
快捷键 ctrl + shift + p 打开命令面板（或者在“查看”菜单中打开命令面板），然后输入 markdown: Open Preview
- 支持将多个文件夹添加到工作空间
在“文件”菜单中，通过“将文件夹添加到工作区”，“将工作区另存为”，“从文件打开工作区”等进行相关操作
- 如果打开了多个终端，可以让 AI 在指定的终端中执行命令
在终端列表中，选择某一个终端，然后点击鼠标右键，选择“标记为 AI 使用”
