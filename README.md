# Vibe Coding Demo


### trae 的 ide 模式和 solo 模式
- ide 模式：类似于助手模式，所有步骤都需要开发者的确认，相当于在传统的开发模式下集成了 ai 助手
- solo 模式：类似于代理模式，所有步骤 ai 都自己做了，中间的过程可以实时监测
- 点击 trae 的左上角可以切换 ide 模式和 solo 模式

### trae 的聊天框
1. 通过 @ 选择智能体，切换至 solo 模式
   1. @SOLO Coder
2. 通过“四角星光按钮”优化提示词
   1. 输入：新建 snake 目录，并在 snake 目录内开发一个基于 html 的贪吃蛇游戏
   2. 点击“四角星光按钮”优化提示词
   3. 发送
3. 通过 # 引用上下文
   1. #snake/todo.md 参考此文件修改游戏逻辑
   2. 运行游戏
4. 切换至 ide 模式，引用图片（solo 模式无法引用图片）
   1. 点击聊天框的“图片按钮”，选择 snake/button.jpg
   2. 按照这个按钮的样式修改游戏中的所有按钮
   3. 运行游戏
5. 在聊天框中支持的快捷键
   1. Enter：发送
   2. Shift + Enter：换行
   3. 上下：切换历史输入


### trae 的小技巧
- 预览 markdown 文件
快捷键 ctrl + shift + p 打开命令面板（或者在“查看”菜单中打开命令面板），然后输入 markdown: Open Preview
- 支持将多个文件夹添加到工作空间
在“文件”菜单中，通过“将文件夹添加到工作区”，“将工作区另存为”，“从文件打开工作区”等进行相关操作
- 如果打开了多个终端，可以让 AI 在指定的终端中执行命令
在终端列表中，选择某一个终端，然后点击鼠标右键，选择“标记为 AI 使用”
